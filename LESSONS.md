2026-07-09 | adversarial review of the four tab handlers found the primary "Research" button could crash the whole page: the annual-report text_provider (NSE auto-fetch AND the manual-URL override) called adapter.fetch (urlopen + pypdf) unguarded, and _extract called text_provider unguarded, so a timed-out / 403 / unparseable PDF (common from a cloud IP) threw a full-page stack trace at the parents | the "returns None if unavailable" contract was documented but not actually implemented — the fetch site had no try/except | honor abstain-on-failure at the boundary: wrapped text_provider in _extract and adapter.fetch in fetch_annual_report_text with try/except -> None; a report just abstains, never crashes. Same review: the Ask-tab verified-fact tick was tier-only (is_verified_fact checks all-PRIMARY) with NO numeric grounding, so an LLM misquote citing a real primary chunk would show a green ✓ — the exact wrong-figure-with-confidence failure the app exists to prevent (latent: unreachable for the parents' real-stock questions today since no real PRIMARY docs are deployed, only news). Added numbers_grounded() to _assemble_result: a FACT whose material number (>=3 digits, digit-normalized exact match) is absent from its cited chunk text is downgraded to UNVERIFIED. Bias is conservative on purpose (a wrongly-flagged true fact only shows "reported, not verified"; a false ✓ never happens). Also fixed reload button st.rerun() wiping its own st.error, and risk tiles showing a fabricated 0.00 on empty history.
